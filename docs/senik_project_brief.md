# Проект: Senik — приложение ментальных моделей

Второй продукт в линейке студии. Первый продукт — Qaizo (финансы), в модерации Google Play. Имя студии и приложения — в процессе выбора.

---

## 1. Концепция

**Что это:** Мобильное приложение с одной ментальной моделью в день. Формат — вертикальный свайп-интерфейс, редакторская подача, интерактивные сценарии.

**Позиционирование:** Инструмент ясного мышления для тех, кто принимает решения. Не «справочник», а «ежедневная тренировка».

**Ниша:** Senik / инструменты мышления / self-improvement knowledge.

**Целевая аудитория:** Предприниматели, продакт-менеджеры, консультанты, founders, MBA, люди 28-50 с интересом к рациональности и саморазвитию.

**Главное отличие от конкурентов (Re:Mind, Mind Models, SnapLearn):**
- Редакторский сторителлинг вместо сухих справочников
- Интерактивные сценарии с голосованием и глобальной статистикой
- Сильный арт-направленный визуал (не стандартный SaaS-дизайн)
- Привязка к живым примерам из реальности
- Блок "действие на сегодня" в каждой карточке

---

## 2. Технический стек

**Mobile:** React Native + Expo SDK 54 (тот же стек, что в Qaizo)
**Auth:** Firebase Auth (Email/Password + Google, Apple)
**База:** Firestore (абстракция через dataService.js по образцу Qaizo)
**Push:** Firebase Cloud Messaging
**Analytics:** Firebase Analytics + сегментация по языку
**Аналитика голосований:** Firestore Counters (через distributed counter pattern)
**Хостинг контента:** Firestore документы + Cloud Storage для иллюстраций
**CI/CD:** Expo EAS Build + EAS Submit

**Причина переиспользования стека Qaizo:** Sean уже знает инфраструктуру, общая кодовая база для paywall/auth/push, быстрый MVP.

### Коллекция `scenario_stats` — глобальная статистика голосований

Используется distributed counter pattern во избежание write contention.

```json
{
  "model_id": "sunk_cost",
  "total_votes": 14840,
  "options": {
    "stay": { "count": 10091, "percentage": 68 },
    "switch": { "count": 4749, "percentage": 32 }
  },
  "last_updated": "2026-04-24T09:15:00Z",
  "min_votes_to_show": 1000
}
```

До достижения min_votes_to_show (1000) в UI показывается плейсхолдер «Ты 147-й, кто ответил. После 1000 ответов увидишь, что выбрали другие».

Для академических моделей (вагонетка, зефир и т.д.) — захардкоженная статистика из реальных исследований, пользовательские голоса не учитываются в основной цифре (можно показать отдельно).

## 4. Дизайн-система

### Типографика
- **Заголовки:** Prata (Google Fonts, serif)
- **Основной текст:** Spectral 400/500/italic (Google Fonts, serif)
- **Моно/лейблы:** JetBrains Mono 400/500/700 (Google Fonts)

### Цветовая система — единая база + акцент по категории

**База (общая):**
```
bg-default: #14100C (warm) или #0E1014 (cool) — зависит от категории
text-primary: #EBDFCC или #E4E7EC
text-dim: #9A8F7E или #8B93A1
text-mute: #6B6258 или #5C6472
border: rgba с прозрачностью
```

**Акцентные палитры по категориям:**
- `cognitive_biases` — янтарь `#E89647` / `#A86E35` (предупреждение)
- `mental_models` — бирюза `#5EEAD4` / `#2A9D8F` (ясность)
- `paradoxes` — фиолетовый `#A78BFA` (загадка)
- `experiments` — коралл `#F87171` (наука)

### Структура карточки (строгий порядок блоков)
1. Top bar: выпуск № · дата · стрик
2. Category label (акцентный цвет)
3. Hook (italic, dim color)
4. Title + subtitle
5. Illustration (SVG, стилизованная под категорию)
6. Body paragraphs (первый с drop cap)
7. Principle callout (левая граница акцентного цвета)
8. Scenario (интерактивный блок с голосованием)
9. Trap warning (опциональный фиолетовый блок)
10. "Где встретите сегодня" — 3 domains
11. Action на сегодня (заливка акцентным цветом)
12. Swipe hint (↑ для следующего уровня глубины)
13. Bottom nav: dots уровней + Save + Share

### Правила визуального языка
- Минимальная плотность, максимум воздуха
- Каждый блок должен быть доступен свайпом/тапом
- Иконки linе-style 1.5px stroke, без заливок
- Углы элементов 2px (не rounded-ful, не острые)
- Фон не чёрный (`#000`), а тёплый/холодный серо-тёмный
- Иллюстрации — композитные SVG с grain texture, в палитре акцента

### Референсные макеты
Два рабочих HTML-прототипа:
- `mental_model_card.html` — Sunk Cost Fallacy (янтарная палитра)
- `mental_model_occam.html` — Бритва Оккама (бирюзовая палитра)

## 7. Контент-пайплайн

### Первый пак — 30 моделей
**Распределение по категориям:**
- 12 когнитивных искажений
- 10 ментальных моделей (Оккам, Парето, первые принципы и др.)
- 4 парадокса (Монти Холл, Симпсон, Зенон, Ахилл)
- 4 мысленных эксперимента (вагонетка, корабль Тесея, зефирный тест)

**Процесс генерации одной карточки:**
1. Мастер-промпт + шаблон (один раз настроить, 2-3 часа)
2. Генерация драфта через Claude (3-5 минут)
3. Вычитка Alex на русском (10 минут)
4. Перевод на английский через AI (2 минуты)
5. Вычитка Sean на английском (5 минут)
6. Загрузка в Firestore (автоматизированный скрипт)

**Итого:** 20-25 минут на пару RU/EN, ~10-12 часов на весь первый пак в 30 моделей.

### Пополнение после релиза
- 2-3 новых модели в неделю
- 30-50 минут работы в неделю на контент

### Мастер-промпт (черновик для разработки)
См. отдельный файл `master_prompt.md` (будет составлен отдельно).

## 12. Риски и оговорки

**Технические:**
- Firestore write contention на популярных голосованиях → distributed counters обязательно
- Offline mode нужен — пользователь может быть в метро без сети
- Размер иллюстраций — SVG compact, под 50KB каждая

**Контентные:**
- AI-галлюцинации на авторах/датах — каждая модель проверяется Alex через Google
- Повторы примеров — 2-3 итерации промпта до стабильного результата
- Культурная нейтральность — примеры не должны быть слишком американоцентричными

**Продуктовые:**
- Retention на 7 день — критическая метрика, если падает ниже 25% — проблема
- Конверсия в платёж после бесплатного трейла — оптимизировать на неделе 4-6

**Юридические:**
- Авторство моделей — всегда указывать источник (Канеман, Мангер, Оккам и т.д.)
- Не присваивать идеи — формулировки свои, но ссылки на авторов обязательны
- Исследовательские данные (вагонетка, зефир) — указывать источник (MIT, Stanford)
